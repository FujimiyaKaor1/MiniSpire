# 主菜单添加"玩法介绍"按钮计划

## 任务概述
在游戏主菜单界面新增一个"玩法介绍"按钮，玩家点击后可查看游戏规则。

## 当前布局分析

### 现有按钮位置
- `menuStartRect{470.f, 430.f, 340.f, 82.f}` - 开始游戏
- `menuExitRect{470.f, 540.f, 340.f, 82.f}` - 退出游戏

### 按钮尺寸
- 宽度：340像素
- 高度：82像素

## 实施步骤

### 1. 添加新的游戏阶段枚举
在 `Phase` 枚举中添加 `Rules` 阶段：
```cpp
enum class Phase { 
    MainMenu, Battle, Reward, Campfire, Shop, Event, DeckEdit, 
    VictoryThanks, Credits, Defeat, Rules  // 新增
};
```

### 2. 添加按钮矩形定义
添加 `menuRulesRect` 成员变量：
```cpp
sf::FloatRect menuStartRect{470.f, 380.f, 340.f, 82.f};   // 调整位置

sf::FloatRect menuExitRect{470.f, 584.f, 340.f, 82.f};    // 调整位置
```

### 3. 添加返回按钮
```cpp
sf::FloatRect backToMenuRect{470.f, 580.f, 340.f, 72.f};
```

### 4. 修改 renderMainMenu() 函数
- 调整三个按钮的垂直位置
- 渲染"玩法介绍"按钮

### 5. 添加 renderRules() 函数
显示游戏规则内容：
- 基本操作说明
- 卡牌类型介绍
- 战斗流程

- 返回主菜单按钮

### 6. 修改 handleEvent() 函数
- 处理"玩法介绍"按钮点击
- 处理Rules阶段的返回按钮点击

### 7. 修改 render() 函数
- 添加Rules阶段的渲染分支
- 添加Rules阶段的BGM（可选，使用主菜单BGM）

### 8. 更新 isValidTransition()
添加Rules阶段的合法转换

## 玩法介绍内容

```
=== 游戏规则 ===

【基本操作】
• 点击卡牌打出
• 点击"结束回合"结束当前回合
• 按 R 键重置当前运行
• 按 T 键切换动画效果

【卡牌类型】
• 攻击牌（红）：造成伤害
• 技能牌（蓝）：获得格挡或效果
• 能力牌（黄）：持续生效的增益

【战斗流程】
1. 抽牌阶段：抽5张牌
2. 出牌阶段：消耗能量打出卡牌
3. 结束回合：弃掉所有手牌
4. 敌人回合：敌人执行意图

【状态效果】
• 力量：增加攻击伤害
• 格挡：减少受到的伤害
• 虚弱：造成的伤害减少25%
• 易伤：受到的伤害增加50%

【遗物系统】
击败精英敌人有概率获得遗物，
提供各种被动增益效果。
```
