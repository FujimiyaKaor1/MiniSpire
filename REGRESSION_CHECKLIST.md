# 回归检查清单

本清单用于每次平衡调整后验证 4 张新增资源卡（放血/盛怒/薪火之源/添柴）的行为一致性。

## 1. 前置准备

- 进入可战斗状态（任意战斗房间）
- 在本局内确保可获得并打出下列卡牌
- 测试基础版与升级版（带 `+`）各至少 1 次

## 2. 放血（基础版）

- 期望：费用 0
- 期望：打出后立刻失去 3 点生命
- 期望：打出后立刻获得 2 点能量
- 期望：非消耗（回合结束进入弃牌堆）

## 3. 放血+（升级版）

- 期望：费用 0
- 期望：打出后立刻失去 3 点生命
- 期望：打出后立刻获得 3 点能量

## 4. 盛怒（基础版）

- 期望：费用 1
- 期望：打出后立刻获得 2 点能量
- 期望：消耗（进入消耗堆）

## 5. 盛怒+（升级版）

- 期望：费用 0
- 期望：打出后立刻获得 2 点能量
- 期望：消耗（进入消耗堆）

## 6. 薪火之源（基础版）

- 期望：费用 2
- 期望：打出后本场战斗获得“每回合开始 +1 能量”
- 期望：下一回合起始能量为 4（若无其它来源）
- 期望：消耗（进入消耗堆）

## 7. 薪火之源+（升级版）

- 期望：费用 1
- 期望：持续效果与基础版一致（每回合开始 +1 能量）

## 8. 添柴（基础版）

- 期望：费用 1
- 期望：打出后消耗“除本卡外”的其余手牌
- 期望：按被消耗手牌数量等量抽牌
- 期望：本卡不消耗（进入弃牌堆）

## 9. 添柴+（升级版）

- 期望：费用 0
- 期望：其余行为与基础版一致

## 10. 边界场景

- 手牌仅 1 张（只含添柴）时：不应额外抽牌，也不应报错
- 抽牌堆为空且弃牌堆有牌时：应回洗后继续抽牌
- 手牌接近上限（例如 9 张）时：抽牌不应超过 10 张上限

## 11. 每轮改动后必跑

- 构建：使用 CMake Tools Build
- 自动测试：`smoke_self_test`
- 结果记录：若失败，附失败场景、卡牌、预期与实际差异
